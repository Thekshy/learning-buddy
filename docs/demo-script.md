# Learning Buddy · 演示录屏脚本(3 分钟版)

> 评审现场 / 课程答辩专用。

## 🎬 总长:3 分钟 | 分镜:7 段

### 段 1:开篇(0:00 - 0:20)
**画面**:浏览器打开 `localhost:3000`,停在登录页。
**口播**:
> "这是我们组的『智能学习伙伴』,一个多智能体 Web 应用,后端用 Java + Spring Boot,LLM 接入 MiniMax M3,向量库用阿里开源的 zvec 做 RAG。下面我从注册到完整学习闭环演示一遍。"

**操作**:注册一个新账号,登录。

### 段 2:目标选择(0:20 - 0:50)
**画面**:登录后进入首页,学科选择卡片(3 门:Python / 高数 / 英语)。
**口播**:
> "我先选 Python,然后点进『装饰器』这个知识点,自评是初学者。"

**操作**:点 Python → 展开知识树 → 选「装饰器」→ 等级选 BEGINNER → 提交。

### 段 3:多 Agent 协作(0:50 - 1:40)【评分核心】
**画面**:聊天主界面,右侧栏出现 "Agent 调用链" 时间线,依次高亮 Planner、Quiz、Recommender 三个 Agent。
**口播**:
> "现在看右侧的 Agent 时间线——这是我们项目的核心亮点:**不是单一 Chatbot**,而是 **Orchestrator 调度三个专职 Agent 协作**。
> - 路径规划 Agent 把『学装饰器』拆成 3 个阶段
> - 练习题 Agent 出了 3 道预测试题摸底
> - 资源推荐 Agent 拉了 5 个教程
> 每个 Agent 的耗时、状态都记录在数据库,前端实时可视化。"

**操作**:用鼠标依次 hover 三个时间线条目,弹出 tooltip 显示耗时。

### 段 4:做练习题(1:40 - 2:10)
**画面**:聊天流中弹出 Quiz 卡片,3 道选择题。
**操作**:故意答错 1 道(展示错题归集)。

### 段 5:复盘 + 雷达图(2:10 - 2:30)
**画面**:切到"进度" Tab。
**口播**:
> "看这个雷达图——6 个维度的掌握度都从 0 涨到了 10% 左右,做错的那道题进了错题本。这是 Reviewer Agent 的功劳。"

**操作**:展开"错题本"标签,展示错题。

### 段 6:RAG 答疑(2:30 - 2:50)
**画面**:返回聊天 Tab,先上传一个 PDF(可提前放在桌面上)。
**口播**:
> "这是 RAG 演示——我上传一份 PDF 课程资料,然后问一个资料里才有的细节问题。"

**操作**:上传 PDF → 等 1 秒(后台切片入库)→ 提问"装饰器的 @wraps 是干嘛的"→ Tutor 返回答案,带引用片段标号 `[1]`、`[2]`。

### 段 7:收尾(2:50 - 3:00)
**画面**:回到 Agent 时间线,展示完整 5 段调用链。
**口播**:
> "一次『选知识点 → 学 → 做 → 答 → 看进度』的完整闭环就完成了。代码已开放在 [GitHub 仓库地址]。谢谢。"

---

## ⚠️ 演示前自检清单

- [ ] `.env` 已填好 LLM_API_KEY
- [ ] `mvn spring-boot:run` 已启动,日志无 ERROR
- [ ] `npm run dev` 已启动,首页能打开
- [ ] 上传用 PDF 已放在桌面,文件名英文(避免编码问题)
- [ ] 浏览器全屏、关掉无关 Tab
- [ ] 网络通畅(LLM 调用需联网;若不可用,`LLM_FALLBACK_MOCK=true` 走离线 mock,仍能演示 Agent 调用链)
